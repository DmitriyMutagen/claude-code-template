Что это за система?

Claude Code Infrastructure — это полный стек конфигурации AI-ассистента для Aragant.group. Система превращает Claude из чат-бота в настоящего CTO: с памятью, правилами, автоматическими ролями, геймификацией и самообучением. Результат — AI, который думает на уровне Principal Engineer и действует как elite full-stack developer.

🧠

CTO-слой — Умный AI-CTO

§1

Claude = CTO. Дмитрий = Founder. Система принудительно думает сверху вниз: стратегия → архитектура → дизайн → код. Запрещён прыжок сразу в код.

Главный принцип
Прежде чем думать КАК — ответь ЗАЧЕМ. Бизнес-цель первична.

Классификация задач (автоматическая)

Мелочь
1 файл, до 5 минут — делать сразу без фаз. Типичные примеры: переименовать переменную, добавить строку в конфиг, исправить опечатку.
Задача
Фича или баг, 1-3 дня. Обязательны Фазы 1 и 2 (стратегия + архитектура), потом код.
Проект
Новый модуль, архитектура, больше 3 дней. Все фазы обязательны. Без согласования архитектуры — стоп.

Фазы работы (0–4)

Фаза 0
Понимание
Интерпретация намерения, классификация масштаба
Фаза 1
Стратегия
Зачем? Для кого? Метрика? Бизнес-цель?
Фаза 2
Архитектура
Модули, интерфейсы, варианты, согласование
Фаза 3
Реализация
Код после ок на архитектуру
Фаза 4
Верификация
Тесты, lint, Sentry, Definition of Done

Фаза 2: Обязательный шаблон документа

Цель:          [что и зачем]
Архитектура:   [модули, связи, data flow]
Структура:     [новые файлы/директории, изменения в существующих]
Зависимости:   [библиотеки, инструменты, MCP, API]
Модели данных: [схемы, миграции если нужно]
API контракты: [эндпоинты, input/output если применимо]
Рекомендация:  [вариант + обоснование]
Риски:         [что может пойти не так + митигация]
DoD:           [конкретные критерии готовности]

3 варианта решения (обязательно для задач >1 дня)

Быстрый
Минимум кода, максимум готового. Когда нужно сдать вчера. Trade-off: технический долг.
Сбалансированный
Рекомендация в большинстве случаев. Баланс скорости и качества.
Идеальный
Оверинжиниринг — только когда реально нужно масштабирование ×100.

Smart Backup System

§13
backup.sh — Умный бэкап
Бэкапит только критические файлы: 968KB вместо 3.9GB полного ~/.claude/. Авто-ротация: последние 10 бэкапов. Хранение в ~/.claude/backups/. Формат: claude-backup-YYYY-MM-DD_HH-MM-SS-label.tar.gz
restore.sh — Точечное восстановление
Восстановление по компоненту: settings, rules, gamification, memory, skills, all. Интерактивный выбор даты. Подтверждение для деструктивных операций.
