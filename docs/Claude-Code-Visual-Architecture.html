Система хуков — детальная карта

Все 7 типов хуков, что они делают и в каком порядке срабатывают

SessionStart 5 хуков · при каждом запуске
memory-restore.sh → GLOBAL_MEMORY + MEMORY.md gamification-boot → XP banner + уровень mcp-health-check → 13/14 статус cl-v2.1-boot → instincts load process-watchdog → zombie kill
UserPromptSubmit 1 хук · на каждое сообщение
universal-preflight → Context7 reminder + Exa + Tier classify
PreToolUse 4 хука · перед каждым инструментом
[Edit|Write] block-secrets → .env/.pem/.key ЗАПРЕЩЕНО [Bash] block-force-push → main/master force ЗАПРЕЩЕНО [Edit|Write] anti-pattern → bare except + raw SQL [*] serena-reminder → activate_project reminder
PostToolUse 6 хуков · после каждого инструмента
[Write|Edit] xp-tracker → +10 XP за файл [Write|Edit] cl-observation → новое наблюдение в JSON [Write|Edit|Bash|Task] metrics-update → commits/files counter [*] auto-doc-memory → MEMORY.md auto-populate [Bash] sentry-check → новые ошибки после деплоя [Bash] auto-xp → git commit message → +XP
Stop 5 хуков · при завершении сессии
quality-sentinel → TODO/FIXME + code quality scan session-summarizer → decay-7d/session-YYYY-MM-DD.md gamification-stop → XP итоги + Telegram auto-checkpoint → session-handoff.md snapshot generate_guide.py → HTML atlas v bump
PreCompact 1 хук · перед сжатием контекста
auto-checkpoint → сохранить прогресс ДО потери контекста
Notification 1 хук · при долгих операциях
notify-long-op → Telegram если операция >30s

Правила ночного агента (ОБЯЗАТЕЛЬНО)
MERGE CONTRACT
Только feature/* и fix/* ветки. НИКОГДА не мерджи в main автоматически
TWO STRIKES
Если тест фейлится 2 раза подряд — СТОП, пометить blocked, перейти к след задаче
RPI Protocol
Research → Plan → Implement. Не кодить вслепую. Читать файлы сначала
NO .env
Никогда не редактировать .env, credentials, секреты. Абсолютный запрет
