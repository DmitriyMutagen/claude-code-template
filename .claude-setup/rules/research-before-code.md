# Research Before Code (ОБЯЗАТЕЛЬНО)

## Когда срабатывает
Любая нетривиальная задача (>1 файла, новая фича, новая интеграция).

## Обязательный порядок
1. **Context7** — проверь актуальную документацию всех используемых библиотек
2. **Поиск готового** — есть ли библиотека/SaaS/npm-пакет который уже делает это? (Exa, Brave Search)
3. **"Купить vs Построить"** — если есть SaaS за $20/мес который решает проблему → предложи его вместо 2 недель разработки
4. **3 варианта** — всегда показывай:
   - 🟢 Быстрый (минимум кода, максимум готового)
   - 🟡 Сбалансированный (рекомендация)
   - 🔴 Идеальный (когда это оверинжиниринг)

## Запрещено
- Писать код без проверки актуальной документации через Context7
- Изобретать велосипед если есть проверенное решение
- Начинать реализацию без согласования подхода с Дмитрием
- Угадывать API — только документация
