# Data Correctness (ОБЯЗАТЕЛЬНО)

## Принцип
Конечный результат = то что видит ПОЛЬЗОВАТЕЛЬ. Не код, не тесты — реальные данные на экране.

## Обязательные проверки

### При работе с API/БД
- Проверять РЕАЛЬНЫЕ данные, не mock
- Сверять количество записей: сколько в БД vs сколько на фронте
- Имена, даты, числа, цены — ВСЕГДА сверять с источником
- Если API возвращает пустой массив — это БАГ, не "нет данных"

### При работе с маркетплейсами
- Количество отзывов/вопросов: сверять через MCP marketplace API
- Имя покупателя: должно быть реальное, не "Покупатель"
- Рейтинг: проверять что совпадает с источником
- Фото товара: проверять что загружаются

### При генерации контента
- Научные ссылки: проверять через PubMed MCP
- Цены: сверять с актуальными данными
- Характеристики товара: сверять с карточкой маркетплейса

### При деплое
- Открыть в браузере и проверить глазами (Playwright screenshot)
- Проверить все страницы на 500/404 ошибки
- Проверить что формы отправляются
- Проверить что данные отображаются корректно

## СТОП-сигналы (блокируют "готово")
- "Всё работает" без скриншота/E2E — НЕ ПРИНИМАЕТСЯ
- "Тесты прошли" без проверки реальных данных — НЕ ПРИНИМАЕТСЯ
- Числа на фронте не совпадают с БД — БАГ
