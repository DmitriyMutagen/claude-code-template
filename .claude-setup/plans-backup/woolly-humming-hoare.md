## Идея MCP-сервера: «Bionovacia MCP»

Отличная идея! MCP-сервер может стать:
1. **Инструментом для Claude Code** — прямой доступ к базе знаний ингредиентов, проверка рецептур, генерация документации
2. **Демонстрацией AI-native подхода** — показать Сколково что платформа нативно интегрируется с AI-ассистентами
3. **Практическим инструментом для команды** — технологи могут через чат спрашивать о совместимости ингредиентов, нормах, рецептурах

**Инструменты MCP-сервера**:
- `search_ingredients` — поиск ингредиентов по названию, группе, эффекту
- `get_ingredient_details` — полная карточка ингредиента с дозировками, взаимодействиями
- `check_compatibility` — проверка совместимости набора ингредиентов
- `optimize_recipe` — подбор оптимального состава по критериям
- `check_compliance` — проверка рецептуры на нормативные требования
- `generate_label` — генерация текста этикетки/маркировки
- `estimate_cost` — расчёт себестоимости рецептуры
- `predict_stability` — прогноз стабильности и срока годности
- `get_regulatory_info` — информация по нормативным документам
- `search_pubmed` — поиск научных публикаций по ингредиенту

## Что можно взять готовое (reverse engineering / open source)

### Базы данных ингредиентов (подтверждено research — ЗОЛОТАЯ ЖИЛА):
- **NP-KG** (GitHub) — готовый knowledge graph натуральных продуктов, МОЖНО ВЗЯТЬ КАК ОСНОВУ
- **FooDB** — 28K+ химических соединений в продуктах питания
- **USDA FoodData Central** — API для нутриентов (public domain, бесплатно)
- **NIH DSLD** — 76K+ этикеток добавок (Dietary Supplement Label Database)
- **PubChem** — химические свойства веществ (REST API, бесплатно)
- **Open Food Facts** — открытая база продуктов питания
- **DrugBank** (пересечения с лекарствами) — частично открыт

### Конкурентная стратегия (из deep research):
> **Такой платформы НЕ СУЩЕСТВУЕТ нигде в мире.**
> Ближайший аналог — PIPA Corp ($100K+/год, enterprise).
> В России — НОЛЬ платформ для БАД-производства.
> Bionovacia = "PIPA-lite" для SME на open-source в 10x дешевле.
> Уникальная комбинация: AI-формулирование + производство + ЕАЭС compliance.

### NLP для нормативных документов (русский):
- **Natasha + Yargy** — rule-based NER (номера ТР ТС, дозировки, названия веществ)
- **Slovnet BERT NER** — из экосистемы Natasha, легче DeepPavlov в 4 раза
- **razdel** — токенизация русского текста

### Оптимизация рецептур (подтверждено research):
- **pymoo** (NSGA-II/NSGA-III) — **ОСНОВНОЙ** для multi-objective оптимизации (pip install pymoo)
- **scipy.optimize** — constrained optimization для простых задач
- **DEAP** — генетические алгоритмы (визуально впечатляет на демо)
- **ProcessOptimizer** — Bayesian optimization параметров процесса (форк scikit-optimize от Novo Nordisk)
- **PuLP** — LP/MIP для минимизации стоимости при ограничениях

### Графы знаний (подтверждено research):
- **Neo4j Community** + **neo4j Python driver** — основная графовая БД
- **NetworkX** — быстрый прототип графа в памяти
- **FoodKG** (foodkg.github.io) — готовый KG: рецепты + ингредиенты + нутриенты (USDA + FoodOn)
- **FoodOn** (foodon.org) — онтология продуктов питания (farm-to-fork)
- **Neo4j GraphRAG** — RAG pipeline поверх графа знаний

### NER для нормативных документов (подтверждено research):
- **Natasha + Yargy** — rule-based extraction (номера ТР ТС, дозировки мг/МЕ, названия веществ)
- **Slovnet BERT NER** — из экосистемы Natasha, F1 лучше DeepPavlov, в 4 раза легче
- Для MVP: regex + Yargy rules для структурированных паттернов, LLM для сложных случаев

### Визуализация графов (фронтенд):
- **D3.js force-directed** — интерактивный граф связей ингредиентов
- **Cytoscape.js** — специализированная визуализация биологических сетей
- **React Flow** — визуализация потоков данных

### Digital Twin (SimPy для MVP, Eclipse Ditto для production):
- **SimPy** — discrete-event simulation производственной линии БАДов
- **SimPy архитектура**: приёмка → взвешивание → смешивание → грануляция → капсулирование → фасовка → контроль
- **FastAPI + WebSocket** — real-time обновления параметров двойника
- **RamanSPy / rampy** — pipeline для спектроскопии (синтетические данные для демо)
- **opcua-asyncio** — будущая интеграция с промышленными контроллерами

### Специфика спортивного питания (из исследования):
- **Текстурное отвердевание (hardening)** протеиновых батончиков — главная проблема shelf-life
- Причина: **макромолекулярная миграция влаги** от белков к углеводам + **реакция Майяра** + белковая агрегация
- Движущая сила: **градиент активности воды (aw)** между доменами батончика
- Наша модель: термодинамика массопереноса + ANN → прогноз на 90-180 дней
- Корректировка: соотношение сывороточный/соевый протеин, гидроколлоиды, осмотическая нагрузка
- **Деградация термолабильных витаминов** при перегреве — use case для closed-loop контроля
- **Конкурентное ингибирование всасывания минералов** — пример антагонизма (Ca vs Fe, Zn vs Cu)
- **Сезонная и партионная вариативность сырья** — причина недостаточности ПИД-регуляторов

### Интеграция Честный ЗНАК (детализация):
```
Платформа Бионовация → True API ЦРПТ:
  1. Заказ кодов маркировки (DataMatrix)
  2. Нанесение на упаковку (интеграция с линией фасовки)
  3. Агрегация криптохвостов (упаковка → короб → паллет)
  4. Ввод в оборот (документ УПД)
  5. Передача между участниками (при контрактном производстве)
  6. Вывод из оборота (продажа конечному потребителю)
  → Всё автоматически, без участия оператора (бенчмарк: TraceLink Russia BAFS)
```

### Промышленные протоколы (pipeline):
```
Digital Twin → OPC UA (opcua-asyncio) → ПЛК/PLC
                                       → SCADA-система
                                       → Modbus (legacy оборудование)
  Передаёт: setpoints (уставки) — скорости шнеков, температуры зон, давление
  Получает: телеметрия — температура, влажность, обороты, ток двигателей
```

### FAIR принципы (обязательно для Сколково):
- **F**indable — каждый ингредиент, рецептура, документ имеет DOI/URI
- **A**ccessible — API для всех данных, GraphRAG для семантического поиска
- **I**nteroperable — FoodOn OWL онтология, стандартные словари
- **R**eusable — лицензирование, metadata, provenance tracking

- Страховые взносы: 15% (вместо 30%)

5. ИИ грант (20-300M) → AI-решения в приоритетных секторах
